# Telegram Toolbox

An **interactive** Python tool that runs with your personal Telegram account
(Telethon / MTProto). Run it and a **numbered menu** appears; pick a number and
type the inputs — no commands to memorize.

It can: delete messages (many ways), back up / export, search & browse, and send & forward.

---

## 1) Install

```powershell
pip install -r requirements.txt
```

## 2) Get api_id and api_hash

1. Open **https://my.telegram.org** and log in with your phone number (the code
   arrives inside Telegram).
2. Go to **API development tools** and create a new application (any name).
3. You get an **api_id** (number) and **api_hash** (long string).

## 3) Configure

Copy `.env.example` to `.env` and fill in your values:

```ini
TG_API_ID=123456
TG_API_HASH=0123456789abcdef0123456789abcdef
TG_PHONE=+201001234567
```

> The first run asks for the login code Telegram sends you (and your 2FA password
> if enabled). After that it stores a session in `telegram_session.session`, so it
> won't ask again.

---

## 4) Run

**Easiest:** double-click **`run.bat`** (sets up the console and opens the menu).

**Or from a terminal:**
```powershell
python telegram_tool.py
```

The menu appears — type the number of the tool you want and follow the prompts.

## The menu

**DELETE TOOLS**
| # | Tool |
|---|---|
| 1 | Delete messages in a **date range** (from/to) |
| 2 | Delete the **last N messages** |
| 3 | Delete **ALL** messages in a chat (asks you to type `DELETE`) |
| 4 | Delete **only my own** messages (all or in a range) |
| 5 | Delete messages containing a **keyword** |

**BACKUP / EXPORT**
| # | Tool |
|---|---|
| 6 | **Export** a chat to a file (`JSON` + readable text) |
| 7 | **Download** all media from a chat into a folder |

**SEARCH / BROWSE**
| # | Tool |
|---|---|
| 8 | **Search** messages by keyword |
| 9 | List / search your **chats** |
| 10 | Chat **statistics** (message count, oldest/newest) |
| 11 | My **account** info |

**SEND / FORWARD**
| # | Tool |
|---|---|
| 12 | **Send** a message (now or **scheduled**) |
| 13 | **Forward / save** messages to another chat or Saved Messages |

**AUTOMATION**
| # | Tool |
|---|---|
| 14 | Set up **scheduled auto-cleanup** (Windows Task Scheduler) |

## Automatic scheduled cleanup (set it and forget it)

This is the "delete old messages from time to time, automatically" feature.

Use **menu option 14**. It asks for the chat, "delete messages older than N days",
for-everyone vs your-side-only, and a daily/weekly time. Then it:

1. Writes a small `auto_cleanup.bat` runner next to the tool.
2. Creates a Windows **Task Scheduler** task (`TelegramAutoCleanup`) that runs it
   on your schedule — no interaction needed.

To remove it later, run option 14 again and choose **remove**.

> The task runs while you are logged into Windows. The first interactive login
> (option-less run once) must already be done so the saved session exists.

You can also run the automated cleanup manually / from your own scheduler:

```powershell
python telegram_tool.py --auto --chat me --older-than 7 --yes
```

(Leave off `--yes` to preview without deleting.)

---

## How to specify a chat

Any tool that asks for a chat accepts:
- `me` -> your Saved Messages
- `@username`
- a numeric `id` (from tool 9)
- **part of the chat name** -> it searches and matches

---

## Important: about deleting

- **Deletion is permanent.** Every delete tool **shows the count first** and asks
  for confirmation before doing anything.
- Before each delete you choose how:
  - **For everyone** -> removed for you **and the other side** (default).
  - **Only for me** -> removed on your side only.
- It's a good idea to **export (tool 6)** as a backup before any bulk delete.
- Dates use your computer's local timezone.

## Project files

| File | Purpose |
|---|---|
| `telegram_tool.py` | The interactive tool (menu) — **this is what you run** |
| `run.bat` | One-click launcher (double-click) |
| `auto_cleanup.bat` | Auto-generated by menu option 14 (the scheduled runner) |
| `.env` | Your secrets (do not share) |
| `requirements.txt` | Required packages |

> `.env` and `*.session` contain sensitive data — they are in `.gitignore` and
> must not be shared.
